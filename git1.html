COMANDOS

El funcionamiento de los siguientes comandos:

  • LS:Es un comando en sistemas Unix y Unix-like (como Linux y macOS) que se utiliza para listar el contenido de un directorio. Es uno de los comandos más básicos y comúnmente utilizados en la línea de comandos,sirve para mostrar los nombres de archivos y directorios en el directorio actual o en el directorio especificado.
  • CD Directorio:Permite moverse entre directorios del sistema. Puedes cambiar de directorio especificando la ruta absoluta desde el directorio raíz o relativa desde tu ubicación actual, en Linux el directorio actual se indica con el signo .
  • CS ..:Este comando nos permite cambiar de directorio, en algunos casos, el solo poner cd nos regresa al directorio “Home”. cd ~: Sin importar donde nos encontremos, con esto nos regresamos al directorio Home.
  • PWD:Sirve para imprimir el "directorio de trabajo actual". pwd equivale a ejecutar cd en un terminal DOS (host de consola de Windows). Es la carpeta o ruta en la que reside la sesión de Bash actual.
  • MKDIR NOMBRE:Crea un directorio o subdirectorio. Las extensiones de comandos, que están habilitadas de manera predeterminada, permiten usar un comando mkdir para crear directorios intermedios en una ruta de acceso especificada.
  • TOUCH NOMBRE:Se creó para cambiar manualmente la fecha y hora de la última modificación y acceso de un archivo, pero también se utiliza muy a menudo para crear archivos nuevos.
  • RM NOMBRE:Es eliminar los archivos en seguimiento del índice de Git. Además, git rm se puede usar para eliminar archivos tanto del índice del entorno de ensayo como del directorio de trabajo.
  • CP NOMBRE DIRECTORIO:Copia ficheros y directorios. Tras el nombre del comando se indicará el fichero a copiar y la ruta y nombre del nuevo fichero. No permite copiar un fichero sobre sí mismo, aunque sí permite sobrescribir un fichero existente
  • MV NOMBRE DIRECTORIO:Es un comando de conveniencia para mover un archivo y luego ejecutar git add en el nuevo archivo y git rm en el archivo antiguo. Sólo mencionamos brevemente este comando en Cambiar el Nombre de los Archivos.

  • GIT --VERSION: Indica la versión del git.
  • GIT CONFIG USER.NAME: Mostrar el nombre de usuario local que esta en Git.
  • GIT CONFIG --GLOBAL USER.NAME "Tu Nombre": Ingresar el nombre de usuario.
  • GIT CONFIG USER.EMAIL: Mostrar el correo electrónico del usuario local en Git.
  • GIT CONFIG --GLOBALUSER.EMAIL "tu correo@ejemplo.com" Ingresar el correo electronico personal para que git lo recnozca.
  • GIT CLONE: Descarga un proyecto y toda su historial de versiones de github
  • GIT ADD INDEX.HTML GIT ADD. :Esto añadirá los archivos indicados al área de staging.
  • GIT STATUS: Enumera todos los archivos nuevos o modificados que se deben confirmar verificar el estado de los archivos en stage
  • GIT COMMIT -M ADD NEW SEARCH FEATURE:Esto creará un nuevo commit en tu repositorio y añadirá una referencia al commit en la rama en la que estás trabajando. Pero antes de hacerlo,tienes que indicar el mensaje del commit.
  • GIT LOG:Enumera el historial de la versión para la rama actual
  • GIT PUSH ORIGIN MAIN: Es un comando que permite enviar los cambios de un repositorio local aun repositorio remoto.
  • GIT REMOT RM ORIGIN:es una parte de un sistema más amplio que se encarga de sincronizar los cambios. Los registros inscritos mediante el comando git remote se utilizan junto con los comandos git fetch, git push y git pull.
